What is HTML?

HTML stands for Hyper Text Markup Language. HTML is the standard for documents thats sole purpose is to be viewed in a web browser.

HTML code is used as the structure of webpages and is described as a webpages skeleton. HTML is often combined with CSS (Cascading Style Sheets) and JavaScript to make aesthetic and dynamic webpages.

What you should already know

Many people use HTML as a starting point in their coding journey. Before exploring HTML you should be aware of the mindset needed for coding and the persistence needed.

There is no requisite for learning HTML. However there are a few things you should be aware of.

  • HTML is only the beginning of understanding Web Development, it is also necessary to understand CSS and basic JavaScript at the very least.
  • You may also want to look into server hosting, web security, web accessibility or even other languages that are used on the web such as PHP.
  • Before beginning you should know what HTML means as well as HTMLs role in making web pages.
Basic Use

HTML is a language that uses tags. Tags surround your variables of information. Below is a p tag (Paragraph) with some text in the middle. Most HTML tag types use both opening and closing tags.

       <p> Hello World! </p>

HTML is also a nested language. This means you can layer tags inside of each other below is an example of this. Note how the indents line up with each other to show a nested objects.

  
      <div> Layer One
            <div> Layer Two </div>
      </div>

HTML is very beginner friendly as a lot of HTML code is repeated and remains consistent between projects.

Below is a standard start of a HTML file.

       
         <!DOCTYPE html>
         <html lang = "en">
         
          <head>
           <meta charset="UTF-8">
           <title>Web Page title</title>
           <link> rel="stylesheet" href="default.css"</link>
          </head>
          
          <body>
            <main>
              <h1>This is the page heading</h1>
              <p>This is text to display</p>
              <img src = image.jpg alt = "image tag demonstration">
            </main>
          </body>
         </html>

There are three main sections of a HTML file.

  • start/top
  • the head
  • and the body

The start of the file consists of the Document Type and the language for the site.

         
           <!DOCTYPE html>
           <html lang = "en">

The head of the html file contains all of the HTML's file information that does not get displayed on the main page, such as the website title and links to other files needed to display the webpage. Other files you often see are are CSS files and JavaScript files. In the example below shown is the meta charset, title, and a css sheet link.

       
          <head>
           <meta charset="UTF-8">
           <title>Web Page title</title>
           <link> rel="stylesheet" href="default.css"</link>
          </head>

The body of the html file contains all of the information that is to be displayed on the page, you can apply styling to HTML code however this is considered poor practice and is the role of a CSS file. When ran the body of the text should be displayed withn your browser as a skeleton.

Below is an example of a HTML files body code. The example includes a heading, body text (within a paragraph tag), an image and the closing html tag to mark the end of the file.

       
         <body>
            <main>
              <h1>This is the page heading</h1>
              <p>This is text to display</p>
              <img src = image.jpg alt = "image tag demonstration">
            </main>
          </body>
         </html>
HTML 5 and Current Standards

HTML5 was the last update to HTML and was published in 2014. The update increased HTMLS scope to suit the needs of multiple browsers and multiple devices of differing screen sizes. The update also had new features to make the websites more acessible. Most would agree that the update also allowed the expansion of websites to allow them to be more dynamic and eye catching.

The main big changes you should know are the following.

  • Local storage on users devices, this is more secure than cookies and has larger storage capabilites.
  • Improved fonts with better fonts, colours and shadows as well as other font effects.
  • Objects can now interact with the users cursor.
  • Improved and better accessible tags such as article, sections and footer tags.
  • Added tags for media such as the video tag and audio tag.
  • Doctype declaration was shortened.
  • Browser based games, audio and video no longer need installed plugins in order to work.
  • 3D effects can now be rendered in browsers using WebGl.
Accessible HTML 5

One of the big motivators for updating HTML 4 was to help make the web accessilbe to everyone.

HTML5 includes specific tags for web designers. These tags help improve web accessiblity as they provide a consistent layout that screen readers can read. Below is an example of the accessible tags.

      
          <nav> Used to mark the navigation area of a site, most commonly a nav bar.</nav>
          
          <main> Used to mark the main area of the website, there should only be one.</main>
          
          <h1> Used to mark Headings, these go 1-6 and display in different sizes. 
          However all styling should still be applied with CSS so headings should be numbered 
          for heading order rather than size preference.<h1>
          
          <section> Sections represent information that needs to be group together, you can have many sections.<section>
          
          <article> Articles represent stand alone text that is different from the main and the sections. 
          It is content that is unrelated to the others.</article>
          
          <aside> Often sidebars, these are additional content containers that are often on the side of web pages.</aside>
          
          <footer> Footers are used for the end of pages, they often contain 
          disclaimer information and copyright information. </footer>
          
          <div> Divs can be used for any other sections you need to make.
          However you should avoid them where possible as they are not best practice.
          They should only be used in situations where the above sections cannot.
          Screen readers cannot interpret a div tags content.</div>

Improved fonts and design capabilities allow better designed dynamic websites, this is especially good for dyslexic users of the web.

The use of alternate text and transcripts can be used on images, video and audio. These not only provide accessibility for those with disabilities but for those on older machines as the alt text displays if the media fails to load.

Things to avoid

Most web languages have depreciated features. Unlike other languages these are not always removed. The reason for this is to keep older websites functioning. Older websites may rely on these depreciated features to function so if they were removed the websites would no longer display correctly.

Within you HTML file avoid any style tags such as line breaks (br) or font tags, this is not the roll of a HTML file and can cause complications and contraindications when you come to add your CSS to the file. All the website styling should be added using CSS.

Always make sure to nest your HTML file correctly. This improves the codes readability and makes it easier to understand. Below is two examples of code. The first is correctly nested code the second is incorrectly nested code. Nested code does not affect code when it is written correctly. This is because in most case blank spaces are ignored when the code is interpretted.

      
        <div> Div 1 
            <div> Div 2 </div>
        </div>
      
      
      
        <div> Div 1 
        <div> Div 2 
        </div>
        </div>

Try to use accessible tags wherever possible. This makes yout site more accessible and consistent. Consistency is key for screen readers as they will not recognise standard divs and will read them all the same and not in the correct order for your site. This will confuse users.
